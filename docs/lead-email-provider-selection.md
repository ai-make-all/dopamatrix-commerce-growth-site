# Lead Email Provider Selection

## Current Phase

- Phase 5-I only selects the provider and environment checklist.
- No real email sending is implemented.
- `/api/leads` still uses the mock destination adapter.
- `LEAD_DESTINATION_MODE` still defaults to `mock`.
- Frontend submit behavior remains unchanged.

## Decision

Recommended v1 provider:

- Resend.

Alternative:

- Cloudflare Email Service.

Second alternative:

- Postmark.

Not recommended for v1:

- SendGrid.
- Mailgun.
- MailChannels.
- HubSpot email.
- Self-hosted SMTP.

## Why Resend First

- Minimal integration path for Cloudflare Pages Function.
- Can be called from a server-side function with `fetch`.
- API key can stay server-side.
- No frontend exposure.
- Suitable for early lead notification.
- Does not require CRM setup.
- Simple enough for Phase 5-J implementation.

## Why Cloudflare Email Service Later

- Good Cloudflare-native direction.
- May reduce third-party dependency.
- Requires Email Service setup and binding configuration.
- Should be evaluated after custom domain and sender identity are ready.
- Not necessary for the first real email smoke test.

## Why Postmark Later

- Strong transactional email option.
- Suitable if lead notification deliverability becomes critical.
- Workers usage should prefer direct `fetch` to the API rather than relying on SDK.
- Slightly more operational setup than Resend for this project stage.

## Domain and Sender Requirements

- Real email sending should not be enabled until sender domain or sender identity is verified.
- Current project still uses `pages.dev` for deployment.
- Custom domain and sending domain should be clarified before production email.
- From address should use a verified domain-controlled address.
- Reply-to address should point to the real business operator.
- Do not use personal Gmail as production sender.

## Recommended v1 Email Shape

Subject draft:

```text
New DopaMatrix Commerce Lead - {pageType} - {market}
```

Body sections:

- Lead ID.
- Received At.
- Submitted At.
- Page Type.
- Page Slug.
- Source.
- Market.
- Language.
- Audience.
- Business Type / Category / Industry.
- Target Market.
- Growth Goal.
- Suggested Path.
- Environment.
- Preview note: This email was generated by a server-side destination adapter.

## What Not To Include

Do not include:

- Full raw payload.
- Full browser user agent.
- IP address.
- Debug-only object.
- Secret tokens.
- Internal environment values.
- Full attribution object unless later approved.
- Any field that is not needed for first human follow-up.

## Failure Policy

- If email send fails, `/api/leads` should return a safe `DESTINATION_ERROR` in real email mode.
- In mock mode, destination should not fail.
- Without queue or persistence, production real email mode should be enabled carefully.
- Later reliability can use Cloudflare Queues, D1, R2, or external CRM.

## Recommendation

Phase 5-J should implement Resend email notification behind a server-side environment flag.

Production default must remain mock until smoke test passes.
